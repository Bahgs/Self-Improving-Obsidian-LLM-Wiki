# Good Pull-Forward Questions

Good pull-forward questions are specific, source-grounded, and useful for future work.

## Example

Source context: a source argues that valuable AI software should not only answer the current question, but also surface the next useful question.

Good questions:

### For Future Agents

- Which existing wiki clusters already behave like active knowledge systems, and which are still passive summaries?
- Which current project ideas would become stronger if they surfaced the user's next useful question instead of only displaying static dashboards?

### For Schema

- Should strategic source notes require a pull-forward section?
- What rubric would distinguish a summary that merely restates a source from one that improves future thinking?

Why these work:

- They are tied to the source's mechanism.
- They connect to the vault's architecture.
- They can lead to concrete schema or project changes.
